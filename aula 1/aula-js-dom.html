Conceito 04 / 12
getElementById
Selecionando um elemento único pelo atributo id
HTML — identificando o elemento
<p id="mensagem"> Texto original </p> <button id="meu-btn"> Clique aqui </button>
↓
JavaScript — selecionando
const el = document.getElementById("mensagem"); console.log(el); // <p id="mensagem"> el.innerHTML = "Fui alterado!";

⚠️ ID não usa # aqui! O # é do CSS. Em JS só passa a string: "mensagem"

// O que é retornado
situaçãoretorno
ID existe na página Elemento HTML
ID não existe null
Dois elementos com mesmo ID Retorna o primeiro ⚠️
// Verificando se existe antes de usar
const el = document.getElementById("mensagem"); if (el !== null) { el.innerHTML = "Existe e foi alterado!"; } else { console.log("Elemento não encontrado"); }

✅ Regra de ouro: IDs devem ser únicos em toda a página. Se você precisa selecionar vários elementos iguais, use classes.

🔬 Experimente no console: document.getElementById("qualquer-id-da-pagina") e veja o que retorna.

Conceito 06 / 12
querySelector & querySelectorAll
Seletores CSS funcionando dentro do JavaScript
querySelector — retorna o primeiro
// Por ID (como #id no CSS) document.querySelector("#titulo"); // Por classe (.classe no CSS) document.querySelector(".destaque"); // Por atributo document.querySelector("input[type='email']"); // Seletor composto document.querySelector("form .erro");
querySelectorAll — retorna todos (NodeList)
// Todos os li com classe "ativo" const ativos = document.querySelectorAll("li.ativo"); // ✅ NodeList tem forEach! ativos.forEach(el => { el.style.fontWeight = "bold"; });
// Comparação completa dos seletores
métodoretornaforEach?
getElementById 1 elemento ou null n/a
getElementsByClassName HTMLCollection ✗
querySelector 1 elemento ou null n/a
querySelectorAll NodeList ✓

🧠 Dica de memorização: querySelector usa a mesma sintaxe do CSS que vocês já conhecem. #id, .classe, tag — tudo igual!

✅ Recomendação prática: use sempre querySelector e querySelectorAll no dia a dia. São os mais flexíveis.

Conceito 07 / 12
Lendo e modificando conteúdo
innerHTML, textContent e value
innerHTML

Lê e escreve o HTML interno de um elemento. Aceita tags HTML.

const el = document .getElementById("caixa"); // ler console.log(el.innerHTML); // escrever com HTML el.innerHTML = "<strong>Negrito!</strong>";

⚠️ Nunca insira dados do usuário com innerHTML — risco de XSS.

textContent

Lê e escreve apenas o texto puro. Ignora tags HTML. Mais seguro.

const el = document .getElementById("caixa"); // ler (só texto) console.log(el.textContent); // escrever (tag vira texto) el.textContent = "<strong>isso vira texto";

✅ Use para exibir mensagens de erro e conteúdo dinâmico.

value

Lê o que o usuário digitou em um input, select ou textarea.

const input = document .querySelector("#nome"); // pegar o que foi digitado const texto = input.value; console.log(texto); // limpar o campo input.value = "";

🎯 É aqui que você busca os dados do formulário para validar.

Quando usar cada um? Exibir HTML dinâmico → innerHTML  |  Texto seguro → textContent  |  Ler formulário → value
Conceito 08 / 12
Eventos onclick
Três formas de reagir ao clique do usuário
Forma 1 não recomendado

onclick inline no HTML

<!-- HTML --> <button onclick= "minhaFuncao()"> Clique </button> // JS function minhaFuncao() { alert("clicou!"); }

Mistura HTML com comportamento. Difícil de manter e testar.

Forma 2 aceitável

Propriedade onclick no JS

<!-- HTML (sem evento) --> <button id="btn"> Clique </button> // JS const btn = document .getElementById("btn"); btn.onclick = () => { alert("clicou!"); };

Melhor que o inline, mas só permite um handler por evento.

Forma 3 recomendado ✓

addEventListener

<!-- HTML --> <button id="btn">Clique</button> // JS const btn = document .getElementById("btn"); btn.addEventListener("click", () => { console.log("evento disparado"); });

Permite múltiplos listeners. HTML e JS separados. Pode remover depois.

Outros eventos úteis: "click"   "input"   "change"   "submit"   "keydown"   "mouseover"
Estudo de caso 09 / 12
Demo 1 — Contador
Codar ao vivo: botões + e − que alteram um número na tela
roteiro de implementação
1

Criar o HTML: dois botões (#mais e #menos) e um span (#valor)

2

Criar a variável de estado: let contador = 0

3

Selecionar os elementos com getElementById

4

Adicionar addEventListener("click") no botão +, incrementar e atualizar o display

5

Repetir para o botão −. Desafio da turma: como não deixar ir abaixo de zero?

🎯 Ponto de aprendizado: a variável contador é o "estado". O DOM é só o reflexo do estado.

// Código final — construir ao vivo
<!-- HTML --> <div> <button id="menos">−</button> <span id="valor">0</span> <button id="mais">+</button> </div> // JavaScript let contador = 0; const display = document.getElementById("valor"); const btnMais = document.getElementById("mais"); const btnMenos = document.getElementById("menos"); btnMais.addEventListener("click", () => { contador++; display.textContent = contador; }); btnMenos.addEventListener("click", () => { if (contador > 0) { // ← desafio da turma contador--; display.textContent = contador; } });
Estudo de caso 10 / 12
Demo 2 — Formulário
Lendo inputs e exibindo mensagens de erro no clique do submit
campos a implementar
campotipovalidação
Nome text Não pode estar vazio
E-mail email Deve conter @
Idade number Maior que 0

Padrão: mensagem de erro aparece num <span> embaixo de cada campo — nunca em alert().

🧠 Estratégia: criar uma função mostrarErro(id, msg) para reutilizar.

// Lógica de validação
<!-- HTML de cada campo --> <input type="text" id="nome"> <span id="erro-nome"></span> <input type="email" id="email"> <span id="erro-email"></span> // JavaScript function mostrarErro(id, msg) { document.getElementById(id).textContent = msg; } document.getElementById("submit") .addEventListener("click", () => { const nome = document .getElementById("nome").value; const email = document .getElementById("email").value; const idade = document .getElementById("idade").value; // Valida nome if (nome.trim() === "") { mostrarErro("erro-nome", "Nome é obrigatório"); } else { mostrarErro("erro-nome", ""); } // Valida e-mail if (!email.includes("@")) { mostrarErro("erro-email", "E-mail inválido"); } });
